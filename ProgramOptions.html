PE_BASH_MAKER.py

Usage:

PE_BASH_MAKER.py --ref REF --r1src R1SRC.fq --r2src R2SRC.fq
 --min MINMEM --max MAXMEM --cut CUTOFF --Ncut NCUT
--rlength RLENGTH --blength BLENGTH --slength SLENGTH
--read_type READ_TYPE --filt FILT [--progInd PROGIND] [--isize ISIZE] [--absolute] [--parallel]

Required arguments:

 

--ref

The reference genome.

 

--r1src,
--r2src

Input fastq files containing the raw sequence data.

 

--min, --max

Equivalent to --minmem, --maxmem used in ConsensusMaker.py, see below. 

 

--blength

Equivalent to --barcode_length used in tag_to_header.py, see below. 

 

--slength

Equivalent to --spacer_length used in tag_to_header.py, see below. 

 

--rlength

Length of the original untrimmed read [101].

 

--cut, --Ncut,
--read_type, --filt

Equivalent to the options of the same name in ConsensusMaker.py, see below. 

Optional arguments:

 

-h

Show a help message and exit.  Useful for entering commands because it shows all the arguments. Should only be entered by itself. 

 

--progInd

Equivalent to --read_out in tag_to+header.py, see  below. 

 

--isize

Equivalent to the command of the same name in ConsensusMaker.py, see below.

 

--absolute

The location entered for the program when running is an absolute position, not a relative position.

 

--parallel

Run alignments in parallel.  Not currently supported. 

 

--sam_tag

The SAM tag that store the duplex tag sequence (can be set one more times). Otherwise use the sequence in the read name.

Required arguments:

 

--infile1,
--infile2

Input fastq files containing the raw sequence data.

 

--outfile1,
--outfile2

Output fastq  files for post-tag_to_header sequences.

 

--barcode_length

Length of the duplex tag sequence. [12]

 

--spacer_length

Length of the spacer sequences used. [5]

Optional arguments:

 

-h

Show a help message and exit.  Useful for entering commands because it shows all the arguments. Should only be entered by itself. 

 

--adapter

Optional: Spacer sequence for filtering on the presence of the spacer. This could be thrown off by low quality scores.

How often you want to be told what the program is doing. Defaults to 1000000.  In most cases, this is higher than the number of SSCS reads. 

 

 

 

 

 

ConsensusMaker.py

Usage:

ConsensusMaker.py --infile INFILE --tagfile TAGFILE --outfile OUTFILE  --minmem MINMEM --maxmem MAXMEM --cutoff CUTOFF --Ncutoff NCUTOFF --readlength READ_LENGTH --read_type READ_TYPE --filt FILT [--isize ISIZE] [--read_out ROUT] [--rep_filt REP_FILT] [--sam_tag SAM_TAG]

Required arguments:

 

--infile

Sorted, paired-end .bam file containing aligned sequences. 

 

--outfile

Output bam file for SSCSs. 

 

--tagfile

output tagcounts file

 

--minmem

Minimum number of reads allowed to comprise a consensus. [3]

 

--maxmem

Maximum number of reads allowed to comprise a consensus. [1000]

 

 

--minmem and --maxmem set the range of family sizes (constrained by cigar string) that can be used to make a consensus sequence.  Examples use --minmem of 3 and --maxmem of 1000

 

 

Example 1:

Ten reads (readlength = 80) have a particular barcode.  Of these ten, nine of them have a cigar string of 80M, while one has a cigar string of 39M1I40M.  Only the nine with a cigar string of 80M are sent on to be made into a SSCS.

 

 

Example 2:

Three reads (readlength 80) have a particular barcode.  Of these, two have a cigar string of 80M, and one has a cigar string of 20M1D60M.  No SSCS results.

 

 

Example 3:

A family with over 1000 members exists.  A random sample of 1000 reads from that family is used to make a SSCS.

 

--cutoff

Percentage of nucleotides at a given position in a read that must be identical in order for a consensus be called at that position. [0.7]

 

 

Example:
--cutoff = 0.7

Four reads (readlength = 10) are as follows:

 

 

 

Read 1: ACTGATACTT

 

 

 

Read 2: ACTGAAACCT

 

 

 

Read 3: ACTGATACCT

 

 

 

Read 4: ACTGATACTT

 

 

 

The resulting SSCS is:

 

 

 

             ACTGATACNT

 

--Ncutoff

With --filt 'n', maximum fraction of Ns allowed in a consensus [1.0]

 

 

Example:
--Ncutoff = .1,
--readlength = 20

Two SSCSs are generated as follows:

     

 

 

SSCS 1: ACGTGANCTAGTNCTNTACC

       

 

 

SSCS 2: GATCTAGTNCATGACCGATA

       

 

 

SSCS 2 passes the N filter (10%) with 1/20 = 5% Ns, while SSCS 1 does not with 3/20 = 15% Ns.

 

--readlength

Length of the input read that is being used. [80]
If this value is set incorrectly, the program will often crash with an error message about sequence length not matching quality score length, or will output an empty SSCS bam file. 

 

--read_type

A string specifying which types of read to consider. ['dpm']

 

 

Options are:

 

 

   

 

 

d:  Paired-end reads where both reads in the pair map, and where the two are properly paired (read 2 maps in the opposite direction and on the opposite strand from read 1). 

 

 

 

Flags are 99, 83, 163, and 147.

  

 

 

p: Paired-end reads where both reads in the pair map, but the two are not properly paired. 

 

 

 

Flags are 97, 81, 161, 145, 129, 65, 177, and 113.

 

 

  

m: Paired-end reads where only one read in the pair maps. 

 

 

 

Flags are 181, 117, 137, 133, 73, 89, 69, and 153.

 

 

   

n: Paired-end reads where neither read in the pair maps, and single end unmapped reads.

 

 

 

Flags are 141, 77, and 4. 

   

 

 

s: Single end mapped reads.

 

 

 

Flags are 0 and 16. 

 

--filt

A string indicating which filters should be implemented.  ['os']’

 

 

Filters are:

 

 

   

 

 

o: Overlap filter. Filters out any read pairs which overlap.  Only works on  reads of type d (see above).

   

 

 

s: Softclipping filter.  Filters out any reads which have been soft- clipped in alignment.  This avoids later problems.

   

 

 

n: N filter. Filters out consensus sequences with a higher percentage of Ns than the threshold imposed by --Ncutoff.  Without this option,
--Ncutoff doesn't do anything. 

Optional arguments:

 

-h

Show a help message and exit.  Useful for entering commands because it shows all the arguments. Should only be entered by itself. 

 

--isize

If not -1, sets the maximum distance between read 1 and read 2 for the two to not be considered unpaired.  Only works if --read_type is 'd' [-1].

 

--rep_filt

Remove tags with homomeric runs of nucleotides of length x. [9]

 

--read_out

How often you want to be told what the program is doing. Defaults to 1000000.  In most cases, this is higher than the number of SSCS reads. 

DuplexMaker.py

Usage:

DuplexMaker.py --infile INFILE --outfile OUTFILE --Ncutoff NCUTOFF --readlength READ_LENGTH --barcode_length BLENGTH [--read_out ROUT] [--gzip-fqs]

Required arguments:

 

--infile

Input .bam file containing sorted SSCSs.

 

--outfile

Output output file name prefix (output FASTQS will be prefix.r1.fq and prefix.r2.fq).

 

--Ncutoff

     Maximum percentage of Ns allowed in a consensus [1.0]

 

 

Example:
--Ncutoff = .1
--readlength = 20

Two DCSs are generated as follows:

       

 

 

DCS 1: ACGTGANCTAGTNCTNTACC

       

 

 

DCS 2: GATCTAGTNCATGACCGATA

     

 

 

DCS 2 passes the n filter (10%) with 1/20 = 5% Ns, while DCS 1 does not with 3/20 = 15% Ns.

 

--readlength

Length of the input read that is being used. [80]
If this value is set incorrectly, the program will often crash with an error message about sequence length not matching quality score length, or will output an empty DCS bam file. 

 

--barcode_length

Length of the barcode sequence. Should match the value in tag_to_header.  [12]

Optional Arguments:

 

-h

Show a help message and exit.  Useful for entering commands because it shows all the arguments. Should only be entered by itself. 

 

--read_out

How often you want to be told what the program is doing. Defaults to 1000000.  In most cases, this is higher than the number of SSCS reads. 

 

--gzip-fqs

Output gzipped fastqs (.gz will be added to the output FASTQs). 

CountMuts.py

Usage:

Cat INFILE.pileup | CountMuts.py d MINDEPTH -C MAX_CLONALITY
-c MIN_CLONALITY -n N_CUTOFF [-s START] [-e END] [-u]

Required arguments:

 

-d

Minimum depth for counting mutations at a site
(default = 20).

 

-C

Upper cutoff of mutant reads for scoring a clonal mutation  (default = 0.1).

 

-c

Lower cutoff of mutant reads for scoring a clonal mutation (default = 0).

 

-n

Maximum fraction of N's allowed to score a position (default = 0.05).

Optional Arguments:

 

-h

Show a help message and exit.  Useful for entering commands because it shows all the arguments. Should only be entered by itself. 

 

-s

Position at which to start scoring for mutations
(default = 0). 

 

-e

Position at which to stop scoring for mutations. If set to 0, no position filtering will be performed (default = 0).

 

-u

Only count one mutation of a given type per position

NOTE: This program is still under development and is not supported.  It can still be used to gain a general idea of bias in N generation and mutations throughout a read, but values may be a bit off. 

Required arguments:

 

-i

An imput file.  If None, defaults to stdin [None].

 

-o

A filename for the output file.  If None, outputs to generates one from the input file name.  Required if -i is not used [None].

 

-l

The length of a single read

 

-C

Maximum clonality to allow when considering a position  [0.1]. 

 

-n

Maximum fraction of N's allowed to score a position (default = 0.05).

Optional Arguments:

 

-h
